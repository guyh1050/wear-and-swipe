# Wear&Swipe — פרויקט אופנה חכמה

אפליקציית אופנה שלומדת את הטעם האישי של המשתמש בעזרת אלגוריתם המלצות
מבוסס וקטורים (Content-Based Filtering + Cosine Similarity), וממליצה
על פריטים ועל שילובי לבוש (אווטפיטים).

## הרצה

הפרויקט בנוי כ-ES6 Modules ורץ ישירות בדפדפן — **ללא התקנות וללא build**.
מכיוון שדפדפנים חוסמים `import` מקבצי `file://`, יש להגיש דרך שרת:

- **GitHub Pages** — פשוט להעלות את התיקייה; הכל יעבוד.
- **הרצה מקומית** — `python3 -m http.server` בתיקיית הפרויקט, ואז לגלוש ל-`localhost:8000`.

## ירושה (Inheritance) ופולימורפיזם

הפרויקט משתמש בירושה בשני מקומות מרכזיים:

### 1. שכבת האחסון
```
StorageProvider (אבסטרקטי)
├── FirebaseStorage       — שומר בענן (Firestore)
└── LocalStorageProvider  — שומר מקומית (localStorage)
```
שתי היורשות מממשות את אותו ממשק (`get`/`set`/`delete`). שאר המערכת
עובדת מול `StorageProvider` ולא יודעת איזו מימוש פועל בפועל — אם
Firebase זמין משתמשים בו, אחרת נופלים אוטומטית לאחסון מקומי.
זוהי הדגמה של **פולימורפיזם** ו**עקרון ההחלפה של ליסקוב**.

### 2. שכבת האלגוריתם
```
Recommender (אבסטרקטי)
├── VectorRecommender  — מדרג פריטים לפי דמיון קוסינוס
└── OutfitBuilder      — בונה שילובי לבוש
```
מחלקת האב מגדירה את `recommend()` המשותפת, וכל יורשת מממשת את
`score()` בדרך שלה. אותה לוגיקת דירוג עובדת לשתיהן (פולימורפיזם).

## האלגוריתם — איך ה-AI עובד

1. כל פריט מתואר ב**וקטור** בן 16 מספרים (מאפיינים: ספורטיבי, אלגנטי...).
2. כשהמשתמש עושה לייק, נבנה **וקטור טעם** = ממוצע משוקלל של הפריטים שאהב
   (סופר-לייק שוקל פי 3).
3. **דמיון קוסינוס** מודד את הזווית בין וקטור הטעם לכל פריט — קרוב ל-1 = התאמה.
4. הפריטים מדורגים, והגבוהים עולים ראשונים.
5. **אווטפיטים** נבנים בשקלול: 40% צבע · 30% וקטור · 30% טעם.

## צד שרת (Backend)

- **Firebase Authentication** — הרשמה והתחברות מאובטחת.
- **Cloud Firestore** — מסד נתונים בענן ששומר לכל משתמש את נתוניו.
- **Security Rules** — כללי הרשאה: כל משתמש ניגש רק לנתוניו; רק המנהל
  (guy) רשאי לערוך את הקטלוג. ראה `firestore-rules.txt`.
